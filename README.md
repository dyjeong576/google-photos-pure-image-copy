# Pure Image Copy

지원 웹사이트에서 이미지를 우클릭해 실제 이미지 데이터로 클립보드에 복사하는 Chrome 확장 프로그램입니다.

Chrome 기본 `이미지 복사`로 복사한 이미지가 카카오톡, 네이버 블로그, 웹 편집기 등에서 제대로 붙여넣어지지 않는 경우를 줄이기 위해 만들었습니다. 설정한 사이트 화면에 표시되는 이미지 리소스를 가져와 브라우저 캔버스에 다시 그린 뒤, `image/png` 형식의 클립보드 이미지로 복사합니다.

## 어떤 프로그램인가요?

이 프로젝트는 기본 지원 사이트와 설정에서 추가한 사이트에서 동작하는 Chrome Manifest V3 확장 프로그램입니다.

기본 지원 사이트는 Google Photos, Google Drive, iCloud, Instagram, Facebook, Notion입니다.

지원 사이트에서 이미지를 열고 그 위에서 우클릭하면 선택한 언어의 이미지 복사 메뉴가 나타납니다. 이 메뉴를 누르면 확장 프로그램이 화면의 이미지 리소스를 읽고, 브라우저 안에서 PNG 이미지로 변환한 뒤, 운영체제 클립보드에 이미지 데이터로 넣습니다.

일반적인 이미지 URL 복사, 링크 복사, HTML 조각 복사가 아니라 붙여넣기 가능한 이미지 데이터 자체를 클립보드에 쓰는 것이 목적입니다.

## 왜 만들었나요?

웹사이트의 이미지는 화면에는 일반 이미지처럼 보이지만, 내부적으로는 로그인 세션이나 별도 이미지 제공 도메인이 필요한 리소스인 경우가 많습니다.

이런 환경에서 Chrome의 기본 `이미지 복사`를 사용하면 다음과 같은 문제가 생길 수 있습니다.

- 붙여넣기 대상 서비스에서 이미지가 보이지 않음
- 이미지가 아니라 링크나 다른 데이터처럼 처리됨
- 네이버 블로그 같은 편집기에서 붙여넣기가 안정적으로 되지 않음
- 카카오톡이나 웹 입력창에 붙여넣을 때 이전 클립보드 내용이 붙음

이 확장 프로그램은 이미지를 다시 캔버스에 그려 `image/png` 클립보드 데이터로 만들어 복사합니다. 즉, 붙여넣기 대상 입장에서는 외부 사이트 링크가 아니라 일반 PNG 이미지가 들어온 것처럼 처리됩니다.

## 주요 기능

- 설정에서 활성화한 사이트에서만 우클릭 메뉴를 표시합니다.
- 확장 아이콘을 클릭해 적용 사이트를 추가, 삭제, 활성화, 비활성화할 수 있습니다.
- 설정에서 영어, 한국어, 중국어 간체, 일본어, 스페인어 중 표시 언어를 선택할 수 있으며 기본값은 영어입니다.
- 이미지 우클릭 메뉴에 선택한 언어의 이미지 복사 항목을 추가합니다.
- 표시된 이미지 리소스를 가져와 PNG 이미지 데이터로 변환합니다.
- 변환된 PNG를 클립보드에 씁니다.
- PNG 용량이 약 20MB를 넘으면 비율을 유지하면서 자동 축소합니다.
- 네이버 블로그의 이미지 붙여넣기 용량 제한에 걸릴 가능성을 줄입니다.
- 복사 진행 중 확장 아이콘 배지에 `...`를 표시합니다.
- 복사 성공 시 확장 아이콘 배지에 `OK`를 표시합니다.
- 복사 실패 시 확장 아이콘 배지에 `FAIL`을 표시합니다.
- 페이지 내부 복사가 실패하면 가능한 경우 offscreen document를 이용해 한 번 더 복사를 시도합니다.

## 사용 대상

이 확장 프로그램은 다음 상황에 특히 유용합니다.

- 지원 웹사이트의 이미지를 네이버 블로그 글쓰기 화면에 자주 붙여넣는 경우
- 웹 이미지를 카카오톡, 웹 메신저, 게시판, CMS 편집기에 빠르게 붙여넣고 싶은 경우
- Chrome 기본 `이미지 복사`가 특정 웹사이트에서 안정적으로 동작하지 않는 경우
- 원본 파일 다운로드 후 다시 업로드하는 과정이 번거로운 경우
- 화면에서 보이는 이미지를 빠르게 복사해 다른 서비스에 붙여넣고 싶은 경우

## 동작 방식

전체 흐름은 다음과 같습니다.

1. 사용자가 설정에서 활성화한 사이트에서 이미지를 엽니다.
2. 이미지 위에서 우클릭합니다.
3. 확장 프로그램의 이미지 복사 메뉴를 클릭합니다.
4. service worker가 클릭된 이미지 URL을 확인합니다.
5. 현재 페이지가 활성화된 사이트인지 검사합니다.
6. 이미지 리소스가 지원되는 형식인지 검사합니다.
7. 현재 탭을 활성화합니다.
8. 페이지 안에 스크립트를 주입해 이미지를 가져옵니다.
9. 이미지를 `canvas`에 그립니다.
10. `canvas` 내용을 PNG Blob으로 변환합니다.
11. PNG가 약 20MB보다 크면 비율을 유지하면서 반복적으로 축소합니다.
12. 최종 PNG Blob을 `navigator.clipboard.write()`로 클립보드에 씁니다.
13. 성공하면 확장 아이콘에 `OK`가 표시됩니다.

페이지 안에서 클립보드 쓰기가 실패한 경우, 확장은 이미지 리소스를 가져온 뒤 `offscreen.html`과 `offscreen.js`를 이용해 클립보드 쓰기를 다시 시도합니다.

## 설치 방법

아직 Chrome Web Store에 배포된 확장 프로그램이 아니므로 개발자 모드에서 직접 로드해야 합니다.

1. 이 저장소를 내려받습니다.
2. Chrome 주소창에 `chrome://extensions`를 입력합니다.
3. 우측 상단의 `개발자 모드`를 켭니다.
4. `압축해제된 확장 프로그램을 로드합니다`를 클릭합니다.
5. 이 저장소 폴더를 선택합니다.
6. 확장 목록에 `Pure Image Copy`가 보이면 설치 완료입니다.

## 사용 방법

1. 확장 아이콘을 눌러 설정 화면을 엽니다.
2. 필요한 사이트 도메인을 추가하고 체크박스를 켭니다.
3. 활성화한 사이트에서 복사할 이미지를 엽니다.
4. 이미지 위에서 마우스 오른쪽 버튼을 클릭합니다.
5. Chrome 기본 메뉴의 `이미지 복사`가 아니라 확장 프로그램의 이미지 복사 메뉴를 클릭합니다.
6. 확장 아이콘에 `...`가 표시되는지 확인합니다.
7. 잠시 후 `OK`가 표시되면 복사가 완료된 것입니다.
8. 네이버 블로그, 카카오톡, 웹 편집기, 메신저 입력창 등에 붙여넣습니다.

## 중요한 사용 팁

- 확장 프로그램의 이미지 복사 메뉴를 클릭한 뒤 `OK`가 뜰 때까지 복사 대상 탭을 떠나지 않는 것이 좋습니다.
- Chrome 클립보드 보안 정책상 현재 문서 포커스가 사라지면 복사가 실패할 수 있습니다.
- 복사 실패 시 확장 아이콘에 `FAIL`이 잠시 표시됩니다.
- 확장 프로그램 코드를 수정한 뒤에는 `chrome://extensions`에서 확장 프로그램 새로고침 버튼을 눌러야 합니다.
- 확장 프로그램을 새로고침한 뒤에는 열려 있던 복사 대상 탭도 새로고침하는 것이 안전합니다.
- 이미지가 완전히 로딩된 뒤 우클릭하는 것이 좋습니다.

## 지원 환경

가장 잘 동작하는 환경은 다음과 같습니다.

- 데스크톱 Chrome
- Chrome Manifest V3 지원 버전
- macOS, Windows, Linux의 최신 Chrome 계열 브라우저
- 기본 지원 사이트 및 사용자가 설정에서 추가한 웹사이트
- 해당 사이트에 로그인되어 있거나 이미지를 정상적으로 열람할 수 있는 상태

테스트와 사용을 가장 권장하는 환경은 데스크톱 Google Chrome입니다.

## 브라우저 호환성

이 확장 프로그램은 Chrome Extension Manifest V3 API를 기준으로 작성되었습니다.

사용하는 주요 API는 다음과 같습니다.

- `chrome.contextMenus`
- `chrome.scripting`
- `chrome.action`
- `chrome.offscreen`
- `navigator.clipboard.write`
- `ClipboardItem`
- `createImageBitmap`
- `canvas.toBlob`

따라서 Chromium 기반 브라우저라도 `chrome.offscreen` 또는 클립보드 이미지 쓰기 API 지원 상태에 따라 동작이 다를 수 있습니다. 안정적인 사용은 Chrome을 권장합니다.

## 권한 설명

`manifest.json`에는 다음 권한이 정의되어 있습니다.

| 권한 | 사용 이유 |
| --- | --- |
| `activeTab` | 현재 탭을 대상으로 스크립트를 실행하기 위해 사용합니다. |
| `clipboardWrite` | 변환된 PNG 이미지를 클립보드에 쓰기 위해 사용합니다. |
| `contextMenus` | 이미지 우클릭 메뉴에 확장 프로그램의 이미지 복사 항목을 추가하기 위해 사용합니다. |
| `offscreen` | 페이지 내부 복사가 실패했을 때 보조 클립보드 처리를 수행하기 위해 사용합니다. |
| `scripting` | 페이지 안에서 이미지 fetch 및 canvas 변환 로직을 실행하기 위해 사용합니다. |
| `storage` | 적용 사이트 목록, 활성화 상태, 언어 설정을 저장하기 위해 사용합니다. |

허용된 host permission은 다음 범위로 제한되어 있습니다.

- `https://photos.google.com/*`
- `https://drive.google.com/*`
- `https://www.icloud.com/*`
- `https://www.instagram.com/*`
- `https://www.facebook.com/*`
- `https://www.notion.so/*`
- `https://*.googleusercontent.com/*`
- `https://*.usercontent.google.com/*`

사용자가 사이트를 추가하면 해당 도메인의 `http`/`https` 접근 권한을 추가로 요청합니다.

## 파일 구조

```text
.
├── manifest.json
├── service_worker.js
├── offscreen.html
├── offscreen.js
├── i18n.js
├── options.html
├── options.css
├── options.js
├── icons/
└──README.md
```

각 파일의 역할은 다음과 같습니다.

| 파일 | 역할 |
| --- | --- |
| `manifest.json` | 확장 프로그램 이름, 버전, 권한, 아이콘, service worker를 정의합니다. |
| `service_worker.js` | 우클릭 메뉴 생성, 적용 사이트 검사, 이미지 복사 메인 로직을 담당합니다. |
| `offscreen.html` | offscreen document를 로드하기 위한 최소 HTML 파일입니다. |
| `offscreen.js` | offscreen document 안에서 이미지 변환 및 클립보드 쓰기를 수행합니다. |
| `i18n.js` | 설정 화면과 우클릭 메뉴의 다국어 문구를 정의합니다. |
| `options.html`, `options.css`, `options.js` | 적용 사이트 설정 화면을 담당합니다. |
| `icons/` | Chrome 확장 프로그램 아이콘 이미지가 들어 있습니다. |
| `README.md` | GitHub 게시용 설명 문서입니다. |
| `read.md` | 상세 설명 보관용 문서입니다. |

## 제한 사항

- 지원 웹사이트에 표시되는 이미지 리소스를 복사합니다.
- 각 서비스의 원본 다운로드 파일을 가져오는 프로그램은 아닙니다.
- 원본 업로드 해상도와 복사 결과 해상도가 항상 같다고 보장하지 않습니다.
- PNG 용량이 약 20MB를 넘으면 자동 축소되므로 이미지가 원본보다 작아질 수 있습니다.
- 클립보드 쓰기는 Chrome 보안 정책과 현재 탭 포커스 상태의 영향을 받습니다.
- 각 사이트의 내부 DOM, 이미지 제공 방식, 보안 정책이 바뀌면 동작이 달라질 수 있습니다.
- 모바일 Chrome 확장 환경은 지원 대상으로 보지 않습니다.

## 문제 해결

### 우클릭 메뉴가 보이지 않을 때

다음을 확인합니다.

1. 현재 페이지가 설정에서 활성화된 사이트인지 확인합니다.
2. 이미지가 열린 상태에서 이미지 위를 우클릭했는지 확인합니다.
3. `chrome://extensions`에서 확장 프로그램이 켜져 있는지 확인합니다.
4. 확장 프로그램을 새로고침합니다.
5. 복사 대상 탭도 새로고침합니다.

### 복사 후 붙여넣어도 이전 내용이 붙을 때

확장 프로그램 복사가 실제로 완료되지 않았을 가능성이 큽니다.

1. Chrome 기본 `이미지 복사`가 아니라 확장 프로그램의 이미지 복사 메뉴를 눌렀는지 확인합니다.
2. 확장 아이콘에 `...` 또는 `OK`가 표시됐는지 확인합니다.
3. `FAIL`이 떴다면 복사 대상 탭을 새로고침한 뒤 다시 시도합니다.
4. 이미지가 완전히 로딩된 뒤 다시 우클릭합니다.

### `Document is not focused` 오류가 날 때

Chrome이 현재 문서에서 클립보드 쓰기를 허용하지 않은 상태입니다.

해결 방법은 다음과 같습니다.

1. 확장 프로그램의 이미지 복사 메뉴를 누른 뒤 복사 대상 탭을 바로 떠나지 않습니다.
2. 확장 아이콘에 `OK`가 뜬 뒤 다른 앱이나 편집기로 이동합니다.
3. 복사 대상 탭을 클릭해 포커스를 준 뒤 다시 시도합니다.
4. 계속 실패하면 복사 대상 탭을 새로고침합니다.

### 네이버 블로그에서 5MB 초과가 뜰 때

확장 프로그램은 PNG가 약 20MB를 넘으면 자동으로 축소합니다.

그래도 5MB 초과가 발생한다면 대상 서비스가 실제 PNG 데이터 외에 추가 처리 비용을 포함해 판단하는 경우일 수 있습니다. 이때는 코드의 제한값을 20MB보다 더 낮게 조정해야 합니다.

조정 대상은 다음 값입니다.

- `service_worker.js` 내부 `maxPngBytes`
- `offscreen.js` 내부 `MAX_PNG_BYTES`

예를 들어 20MB 대신 4.5MB 또는 4.0MB로 낮출 수 있습니다.

### 이미지가 흐릿하거나 작아 보일 때

이미지가 자동 축소된 상태일 수 있습니다.

이 확장 프로그램은 붙여넣기 성공률과 용량 제한 회피를 우선합니다. 더 높은 해상도를 유지하려면 용량 제한값을 높일 수 있지만, 네이버 블로그 같은 서비스에서 다시 용량 초과가 발생할 수 있습니다.

## 개발 및 문법 확인

별도 빌드 과정은 없습니다. 정적 Chrome 확장 프로그램 파일을 그대로 로드합니다.

JavaScript 문법 확인은 다음 명령으로 할 수 있습니다.

```bash
node --check service_worker.js
node --check offscreen.js
```

확장 프로그램 수정 후 확인 순서는 다음을 권장합니다.

1. 파일 수정
2. `node --check service_worker.js`
3. `node --check offscreen.js`
4. `chrome://extensions`에서 확장 프로그램 새로고침
5. 복사 대상 탭 새로고침
6. 실제 이미지 우클릭 복사 테스트

## 개인정보 및 보안

이 확장 프로그램은 서버를 사용하지 않습니다.

이미지 처리는 사용자의 브라우저 안에서 수행됩니다. 확장 프로그램은 지원 웹사이트와 관련 이미지 리소스에서 이미지를 읽고, 브라우저 캔버스에서 PNG로 변환한 뒤, 사용자의 클립보드에 씁니다.

다만 지원 웹사이트에 접근하기 때문에 확장 권한은 필요한 범위로 제한되어야 합니다. 현재 host permission은 기본 지원 사이트와 관련 이미지 리소스 도메인으로 제한되어 있습니다.
